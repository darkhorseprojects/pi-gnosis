# Manim video contract

## Scope

Use this skill inside `pi-gnosis` video/lecture routes. The `manim-lecture` graph owns project creation, preflight, render status, and artifact reporting.

## Project structure

```text
<configured manimRoot>/<slug>/
  plan.md
  scene_spec.json
  script.py
  render.sh
  README.md
  media/              # generated by Manim
```

The project root comes from `gnosis_config.paths.manimRoot` plus a deterministic slug.

## Pipeline

1. Define one narrative arc, prerequisite, misconception/trap, visual metaphor, aha moment, and scene list.
2. Write `scene_spec.json` with scene ids, learning objective, visual elements, narration beats, and estimated duration.
3. Write `script.py` with concrete renderable Scene classes and no renderable abstract/base scene.
4. Write executable `render.sh` with explicit scene class order.
5. Preflight: Python syntax, `manim --version`, `ffmpeg -version`, LaTeX only when MathTex is present.
6. Render a draft when requested and preflight passes.
7. Produce an artifact manifest listing project root, final video path if rendered, scene clips, render command, and blockers.

## Visual contract

- one visual objective per scene
- sparse text
- readable labels outside crowded geometry
- consistent camera framing around active objects
- line weights visible at draft quality
- colors carry stable meaning across scenes
- pause after each reveal
- no decorative motion without an instructional role

## Audio extension point

When audio is requested, add graph resources for transcript/script, TTS generation, audio file manifest, subtitle/caption manifest, and Manim audio attachment. Report audio blockers separately from video blockers.
